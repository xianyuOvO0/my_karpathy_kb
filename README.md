\# Karpathy 风格长上下文知识库系统



> 基于长上下文 LLM 的个人知识库构建工具 | 作业项目：物联网/计算机专业大三



本项目严格遵循 Andrej Karpathy 提出的个人知识库新范式：\*\*无向量检索、长上下文 LLM 直接阅读、Markdown 作为唯一存储格式\*\*。



\- \*\*原始资料\*\* → `raw/` 文件夹（零预处理）

\- \*\*自动编译\*\* → LLM 生成结构化 Wiki（实体独立页面、双向链接、标签索引）

\- \*\*可视化\*\* → 使用 Obsidian 打开知识图谱

\- \*\*问答\*\* → 长上下文问答，支持跨文档综合与来源溯源

\- \*\*健康检查\*\* → 自动检测死链、孤立页面



\## 系统架构



```text

┌────────────────────────────────────────────────────────────────┐

│  Stage 1: Ingest (数据摄入层)                                   │

│  └── 原始资料 → raw/目录 (PDF/网页/笔记/docx等)                 │

└────────────────────────────────────────────────────────────────┘

&#x20;                             ↓

┌────────────────────────────────────────────────────────────────┐

│  Stage 2: Compile (知识编译层) ⭐ 核心创新                       │

│  └── LLM读取全部raw/ → 生成结构化wiki/                          │

│  └── 自动提取：实体、概念、摘要、双向链接\[\[concept]]              │

└────────────────────────────────────────────────────────────────┘

&#x20;                             ↓

┌────────────────────────────────────────────────────────────────┐

│  Stage 3: Obsidian IDE (可视化层)                               │

│  └── 使用Obsidian打开wiki/目录，享受图谱视图、反向链接            │

└────────────────────────────────────────────────────────────────┘

&#x20;                             ↓

┌────────────────────────────────────────────────────────────────┐

│  Stage 4: Query (查询问答层)                                    │

│  └── 用户提问 → LLM阅读整个wiki → 生成答案（支持引用溯源）        │

└────────────────────────────────────────────────────────────────┘

&#x20;                             ↓

┌────────────────────────────────────────────────────────────────┐

│  Stage 5: Output Filing (输出归档层)                            │

│  └── 问答结果 → 可选保存为新wiki页面，知识库自我增强             │

└────────────────────────────────────────────────────────────────┘

&#x20;                             ↓

┌────────────────────────────────────────────────────────────────┐

│  Stage 6: Lint (健康检查层)                                     │

│  └── 定期扫描wiki：死链检测、矛盾识别、孤立页面发现               │

└────────────────────────────────────────────────────────────────┘
